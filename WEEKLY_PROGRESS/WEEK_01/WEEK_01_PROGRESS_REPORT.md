# Challenges Encountered

## Git Remote Synchronisation Conflict

Encountered a Git push conflict due to the remote repository already containing README and licence files generated by GitHub during repository creation.

### Resolution

Resolved the issue by:
- pulling remote changes using `--allow-unrelated-histories`
- merging remote and local histories
- successfully pushing the combined repository state to GitHub

This improved understanding of Git remote synchronisation and repository merging workflows.
